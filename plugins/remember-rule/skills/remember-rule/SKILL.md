# remember-rule

앞으로의 Claude 행동을 규정하는 규칙·선호·지침을 사용자가 남기려 할 때 사용.

## 트리거 발화

- "기억해", "기억해둬", "기억해라"
- "저장해", "기록해", "추가해"
- 특정 파일명·저장소를 지목하며 추가/기록하라는 발화 (예: "X에 추가해", "Y에 써")
- "앞으로 ~해", "다음부턴 ~", "앞으로는 ~"

모호한 발화("이거 기억해")는 직전 대화 맥락에서 **무엇을** 기억해야 할지 추론한다.

## 저장 위치 판단

**하드코딩된 매핑 금지.** 매번 내용의 성격·범위를 읽고, 환경에 실제로 존재하는 저장소 중 어디가 어울릴지 판단한다.

판단 기준:
- **범위**: 모든 프로젝트 / 특정 프로젝트 / 특정 도구·환경
- **공유 대상**: 개인 / 팀 전체
- **성격**: 추상 원칙 / 구체 규칙 / 단순 사실·선호

이 기준으로 현재 환경에 존재하는 저장소(전역 AI 지침, 프로젝트별 메모리, 프로젝트 공유 지침, 도구 설정 등) 중 가장 어울리는 곳을 고른다. 구체 경로·파일명은 환경마다 다르니 탐색해서 확인.

사용자가 특정 파일명을 지목해도 그 파일이 정답이라는 보장은 없다. 내용 성격에 맞는 곳이 따로 있으면 제안하고 확인받기. 외부에서 가져와 수정 금지인 파일이 있을 수 있으니 (맥락에 "이 파일 건드리지 마라" 지시가 있었는지) 사전 확인할 것.

## 본문 다듬기

사용자 발화를 그대로 붙여 넣지 말 것. 규칙 문장으로 정리하되 과축약 금지 (짧게 만들다 의미 흐려진 전례 있음).

- 기본 한 줄 불릿. 부연 필요하면 서브불릿 2-3개.
- "왜"가 중요한 규칙은 이유 짧게 병기.
- "반드시", "절대" 같은 수식어는 꼭 필요할 때만.

## 섹션 배치 (구조 있는 파일의 경우)

1. 파일 전체를 읽어 현재 섹션 구성 파악
2. 내용에 어울리는 **기존 섹션**이 있으면 그 아래 추가
3. 없으면 **새 섹션 제목 제안** (사용자 OK 받기)

## 확인 프로토콜

**저장 전 반드시** 다음을 한 번에 제시하고 OK 받기:

1. **무엇을 저장**: 다듬은 규칙 문장
2. **어디에 저장**: 파일 경로 + 섹션 위치
3. **어떤 형태로**: 추가될 불릿/섹션 실제 모양

저장 위치가 애매하면 후보 2개 제시하고 고르게 할 것.

## 하지 말 것

- 사용자 발화 그대로 복붙
- 저장 전 확인 생략
- 과한 압축으로 의미 훼손
- 기존 섹션 있는데도 중복 섹션 신설
