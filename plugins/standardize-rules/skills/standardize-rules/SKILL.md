# standardize-rules

**원칙: 룰 본문은 `AGENTS.md`에만, 다른 파일은 그곳을 가리키게.**

## 발동 분기

영속 지침이 아닌 입력은 거른다:

- 단발성 메모/회고 ("이거 기억해", "메모리에 저장") — 별도 메모리 시스템 담당
- 일회성 결정·맥락 ("이번엔 ~", "지금만 ~") — 영속 룰 아님

## 작업 흐름

1. [스코프 판단](#스코프-판단) → 사용자 확인
2. [SOT 파일 위치 결정](#sot-파일-위치-결정) → 사용자 확인 (첫 실행 또는 위치가 비표준일 때만)
3. [다리 파일 점검](#다리-파일-점검) → 누락·비표준이면 사용자에게 제안 후 보강
4. [룰 본문 추가·갱신](#룰-본문-추가갱신)
5. [결과 보고](#결과-보고)

## 스코프 판단

룰 내용·맥락을 보고 **글로벌 / 프로젝트** 중 어디 적합한지 판단해 사용자에게 확인받는다.

| 신호 | 추천 스코프 |
| --- | --- |
| 특정 레포·코드베이스·도메인 언급 | 프로젝트 |
| 도구 사용 습관, 답변 스타일, 보편적 코딩 원칙 | 글로벌 |
| 사용자가 "이 프로젝트", "여기서만" 등 한정 | 프로젝트 |
| 사용자가 "항상", "어디서나", "모든 작업" 등 | 글로벌 |

## SOT 파일 위치 결정

### 글로벌

표준 후보를 순서대로 탐색:

1. 현재 에이전트의 글로벌 진입 파일(예: Claude Code의 `~/.claude/CLAUDE.md`)에 `@<path>` 임포트 문이 있으면 그 경로를 SOT로 본다.
2. 없으면 사용자에게 위치를 묻는다. 권장 기본값: `~/.claude/AGENTS.md`.
3. 사용자 확정 후 진행. 이후 호출에서는 1번이 항상 작동.

### 프로젝트

표준 위치는 **레포 루트의 `AGENTS.md`**. 그대로 사용한다. 레포 루트는 `git rev-parse --show-toplevel`로 확인.

## 다리 파일 점검

현재 에이전트가 SOT를 읽도록 **표준 연결**이 갖춰져 있는지 확인. 없으면 사용자 확인 후 만든다.

### Claude Code

- 글로벌: `~/.claude/CLAUDE.md`가 SOT를 `@<path>`로 임포트해야 함
- 프로젝트: 레포 루트의 `CLAUDE.md`가 `@AGENTS.md`를 임포트해야 함
- 본문 복붙 금지 (SOT 깨짐)

기존 CLAUDE.md가 본문을 직접 갖고 있고 AGENTS.md가 비어 있으면, 사용자 확인 후 본문을 AGENTS.md로 옮기고 CLAUDE.md를 `@AGENTS.md` 한 줄로 축약하는 마이그레이션을 제안한다.

### 다른 에이전트

발동 환경(현재 켜진 에이전트)이 Claude Code가 아닌 경우 또는 사용자가 다른 에이전트도 같이 쓴다고 명시한 경우에만 다룬다. 그 외엔 건드리지 않는다.

임포트 매커니즘이 없는 도구(예: Cursor의 `.cursor/rules`)는 다음 중 하나를 사용자에게 제안:

- (a) AGENTS.md 본문을 동기화 복사 (SOT는 여전히 AGENTS.md, 다리 파일은 자동 생성된 사본임을 헤더에 명시)
- (b) 해당 도구는 지원 대상에서 제외

표준이 모호한 경우엔 추측 금지 — 사용자에게 묻는다.

## 룰 본문 추가·갱신

기존 AGENTS.md의 **섹션 구조와 톤**을 먼저 학습한 뒤 적합한 위치에 추가.

- 비슷한 룰이 이미 있으면 **업데이트**(중복 항목 추가 금지)
- 신규면 적합한 섹션 하단에 추가. 적합한 섹션이 없으면 새 섹션을 만들되, 기존 헤더 스타일을 따른다
- 룰 본문은 사용자가 한 말의 핵심 의도를 보존하되, 기존 문서 톤(존댓말/평어, 명령형/서술형)에 맞춰 다듬는다
- WHY가 비자명하면 한 줄 사족을 붙인다. 자명하면 본문만

작성 후 diff를 사용자에게 보여주고 확정받는다.

## 결과 보고

대화창에만 짧게:

```
## 룰 추가 완료

- 스코프: <글로벌/프로젝트>
- SOT: <AGENTS.md 경로>
- 추가된 섹션: <섹션명>
- 다리 처리: <변경 없음 / CLAUDE.md @import 추가 / 기타>
- 추가 내용: <한 줄 요약>
```

## 운영 메모

- AGENTS.md가 처음 생성되는 경우 + 외부 임포트가 처음 추가되는 경우, Claude Code는 첫 세션에서 승인 다이얼로그를 띄운다. 사용자에게 미리 안내한다.
- 작업 직전 [code.claude.com/docs](https://code.claude.com/docs) 등 공식 문서로 임포트 문법·진입 파일 위치를 다시 확인한다. 기억으로 박지 않는다.
