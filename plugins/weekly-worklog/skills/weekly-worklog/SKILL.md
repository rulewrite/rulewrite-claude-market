# weekly-worklog

업무 단위(PR 또는 지라 이슈) 로그를 노션에 쌓는다. 노션 본문 읽기·append는 `notion-worklog-writer` 서브에이전트에 위임 — 메인은 트리거·작업 정보 확보·보고만 담당.

## 기본 흐름

1. `~/.claude/state/weekly-worklog/target.json` 로드 → 없으면 [첫 실행](#첫-실행).
2. [작업 정보 확보](#작업-정보-확보).
3. `notion-worklog-writer`에 위임 (입력 명세는 에이전트 정의 참조).
4. 결과 보고서 수신.
5. `format_hint` 갱신이 포함됐으면 `target.json`에 반영.
6. 사용자에게 [짧게 보고](#결과-보고).

## 작업 정보 확보

기본은 작업 1건 = 로그 1건. PR URL/번호 또는 지라 URL/키를 받으면 그걸로. URL 없이 "방금 한 X 작업 써둬"처럼 자유 입력으로 설명해줘도 된다.

- PR: `gh pr view <url> --json number,title,url,body,mergedAt,baseRefName,headRefName,commits`
- 지라: Jira MCP의 `jira_get_issue`

수집:
- 제목, URL/키, 완료 시점 (PR `mergedAt`, 지라 resolution date)
- PR ↔ 지라 상호 링크 (브랜치명·본문에 지라 키가 있거나, 지라 이슈에 PR이 달려 있으면)
- 한 줄 요약

## 대상 문서 관리

대상 노션 문서는 첫 실행 때 한 번 확정해 저장. 매 실행 검색은 토큰 낭비·오탐 위험.

저장 경로: `~/.claude/state/weekly-worklog/target.json`

```json
{
  "kind": "page | database",
  "id": "<notion id>",
  "url": "<notion url>",
  "title": "<문서 제목>",
  "format_hint": "<포맷 요약 메모 — 에이전트가 갱신>"
}
```

### 첫 실행

`target.json`이 없거나 사용자가 "워크로그 문서 다시 정해줘" 류로 재설정을 요청한 경우. 사용자 확인이 필수라 메인에서 처리.

1. 사용자에게 문서 URL 또는 제목·키워드를 묻는다.
2. URL을 주면 그대로 확정. 키워드면 노션에서 검색해 후보를 나열한다.
3. 후보가 여럿이면 사용자가 고르도록 한다.
4. 확정 후 위 경로에 저장. `format_hint`는 비워두고, 첫 위임 후 에이전트가 채워 보낸 값을 반영한다.

## 애매한 경우

- 기록 대상이 불분명하거나 소스가 비면 위임하지 말고 사용자에게 확인.
- 에이전트가 "중복으로 스킵됨"을 보고하면 사용자에게 알리고 수정/스킵/강제 추가 중 선택받는다.
- 에이전트 보고서에 "확인 필요 사항"이 있으면 사용자에게 노출해 답을 받고 필요 시 재위임.

## 결과 보고

대화창에만 짧게.

```
## 워크로그 기록 완료

- 문서: <제목> (<URL>)
- 위치: <엔트리가 들어간 섹션/로우 식별자>
- 추가 항목: <한 줄 요약>
```

## 운영 메모

노션 MCP가 환경에 준비돼 있어야 한다. 없으면 먼저 사용자에게 설치·인증을 안내한다.
