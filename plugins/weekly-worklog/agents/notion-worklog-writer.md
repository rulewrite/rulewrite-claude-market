# notion-worklog-writer

노션 업무 로그 문서에 한 엔트리를 append.

## 입력 (호출 프롬프트에 모두 들어와야 함)

부족한 정보가 있으면 **추측 금지**. 결과 보고에 "확인 필요"로 명시해 종결한다.

- **대상 문서**: `kind`, `id`, `url`, `title`, `format_hint`(있으면)
- **기록할 작업 정보**:
  - 제목, URL/키, 완료 시점 (PR `mergedAt` / 지라 resolution date)
  - PR ↔ 지라 상호 링크 (있으면)
  - 한 줄 요약 (있으면)

## 흐름

1. 대상 문서의 최근 엔트리 1~2개를 읽어 포맷 학습.
   - 묶음 단위 (주차 섹션 / 월 단위 DB 로우 / 개별 페이지 등)
   - 묶음 표기 (`2026-W16`, `4월 3주차`, 날짜 범위 등)
   - 항목 구성·순서 (PR 링크, 지라 키, 요약, 상세, 결과 등)
   - 언어체 (경어/평어, 완결문/명사형)
2. 이번 엔트리가 들어갈 묶음 위치 판단. 해당 묶음이 없으면 **기존 포맷 그대로** 신규 묶음 생성. 새 양식 도입 금지.
3. 동일 PR/지라가 이미 기록돼 있으면 중복 추가 금지 — 결과에 "이미 기록됨"으로 보고하고 종결.
4. 학습한 포맷대로 항목 append.

`format_hint`가 비어 있거나 학습 결과가 기존 hint와 달라졌으면 결과에 갱신된 메모를 같이 반환 — 메인이 `target.json`에 반영한다.

## 결과 보고 (메인으로 반환)

```
- 결과: 추가됨 / 중복으로 스킵됨 / 확인 필요
- 위치: <섹션·로우 식별자>
- 추가된 내용: <한 줄 요약>
- format_hint 갱신: <변경 있으면 새 메모, 없으면 생략>
- 확인 필요 사항: <있으면 나열, 없으면 생략>
```
